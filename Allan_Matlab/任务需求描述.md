# OCXO Allan 稳定度真实采集链路对接说明

## 1. 目标

当前 MATLAB 脚本已经能够完成 Allan 偏差计算、CSV 导入、理论仿真、测量链误差建模和 PNG 输出。

下一阶段工作重点不是 MATLAB 算法，而是建立真实 OCXO 数据采集链路，生成可信 CSV，使后续 Allan 分析结果可以支撑论文中的“实测 OCXO 稳定度”或“基于 FPGA 测量链的 OCXO 稳定度观测”。

核心任务：

```text
真实 10 MHz OCXO 输出
→ 测量链路采集
→ 长时间连续数据记录
→ 可信 CSV 文件
→ MATLAB Allan 偏差分析
```

## 2. 关键矛盾

Allan 偏差实测不是单独测 OCXO，而是测：

```text
OCXO + 参考源 + FPGA/计数器 + 量化误差 + 抖动 + 电源/温度环境
```

因此实测 Allan 曲线满足近似关系：

```text
σ_meas²(τ) ≈ σ_ocxo²(τ) + σ_ref²(τ) + σ_instrument²(τ)
```

若要声称“OCXO 真实 Allan 稳定度”，必须满足：

```text
σ_ref(τ) << σ_ocxo(τ)
σ_instrument(τ) << σ_ocxo(τ)
```

否则只能表述为：

```text
基于当前 FPGA 测量链的 OCXO 输出稳定度观测结果
```

## 3. 当前最大限制

如果使用 50 MHz FPGA 参考时钟直接时间戳 10 MHz OCXO 单周期：

```text
10 MHz 周期 = 100 ns
50 MHz tick = 20 ns
单周期仅 5 个 tick
```

这会导致短 τ Allan 偏差主要由时间戳量化误差决定，而非 OCXO 本体决定。

因此 50 MHz 粗计数链路可以用于：

```text
验证 Allan 算法流程
验证 LabVIEW/FPGA 运算 IP
说明粗计数量化瓶颈
```

但不建议直接用于声称：

```text
高性能 OCXO 的真实短期 Allan 稳定度
```

## 4. 推荐采集方案优先级

### 方案 A：高分辨率时间误差采集，最推荐

目标 CSV：

```csv
sample_index,time_s,time_error_s
0,0.000,0
1,0.001,1.23e-10
2,0.002,1.40e-10
```

说明：

- `time_error_s` 是 OCXO 相对于参考源的相位时间偏差；
- 单位为秒；
- 该格式最适合 MATLAB 直接计算 overlapping Allan deviation；
- 推荐采样间隔 `tau0 = 1 ms`、`10 ms`、`100 ms` 或 `1 s`，根据采集能力决定。

推荐硬件链路：

```text
OCXO 10 MHz
→ 相位比较器 / TDC / 高分辨率时间间隔测量模块
→ FPGA 或上位机
→ CSV
```

### 方案 B：门控频率计数，工程可行

目标 CSV：

```csv
sample_index,time_s,gate_time_s,count,frequency_hz
0,0,1,10000000,10000000.000
1,1,1,10000001,10000001.000
```

说明：

- 使用 1 s、10 s 或更长 gate time；
- 可用于中长期 Allan 偏差；
- 短 τ 分辨率不如 TDC；
- 适合论文中展示工程系统稳定性趋势。

### 方案 C：FPGA 粗时间戳，仅适合算法验证

目标 CSV：

```csv
edge_index,timestamp_count,ref_clock_hz
0,123456789,50000000
1,123456794,50000000
2,123456799,50000000
```

说明：

- 若 `ref_clock_hz = 50 MHz`，时间分辨率为 20 ns；
- 不适合高性能 OCXO 短稳测量；
- 可以用于验证 FPGA 时间戳链路和 Allan 计算流程。

## 5. 推荐 CSV 标准格式

建议最终统一为以下格式，便于 MATLAB 脚本兼容。

### 首选格式

```csv
sample_index,time_s,time_error_s
0,0.000,0
1,0.001,1.23e-10
2,0.002,1.40e-10
```

字段定义：

```json
{
  "sample_index": "样本编号，从 0 或 1 开始，必须连续",
  "time_s": "采样时刻，单位 s",
  "time_error_s": "OCXO 相对参考源的时间误差/相位时间偏差，单位 s"
}
```

### 可接受格式

```csv
sample_index,time_s,frequency_hz
0,0,10000000.001
1,1,9999999.998
```

字段定义：

```json
{
  "frequency_hz": "在 gate time 内测得的 OCXO 输出频率，单位 Hz"
}
```

### 粗计数格式

```csv
sample_index,timestamp_count,ref_clock_hz
0,123456789,50000000
1,123506789,50000000
```

字段定义：

```json
{
  "timestamp_count": "FPGA 参考时钟计数值",
  "ref_clock_hz": "计数参考时钟频率"
}
```

## 6. 采集时长要求

Allan 偏差最大可信 τ 不应接近总数据长度。

建议：

```text
总采集时长 ≥ 10 × 最大分析 τ
```

示例：

| 目标最大 τ | 最少采集时长 | 建议采集时长 |
|---|---:|---:|
| 1 s | 10 s | 100 s |
| 10 s | 100 s | 1000 s |
| 100 s | 1000 s | 2-4 h |
| 1000 s | 10000 s | 12-24 h |

如果只采集几毫秒，不应解释长 τ Allan 偏差。

## 8. 数据质量检查要求

CSV 交付前建议检查：

```text
1. sample_index 是否连续
2. time_s 是否单调递增
3. 是否存在 NaN、Inf、空值
4. 是否存在计数器回绕
5. 是否存在异常跳变
6. 是否存在数据中断
7. 实际采样间隔是否稳定
8. 总时长是否达到目标 τ 的 10 倍以上
9. 参考源稳定度是否优于 OCXO
10. 采集前 OCXO 是否充分预热
```

## 9. MATLAB 脚本对接要求

MATLAB 侧希望拿到以下其中一种文件：

```text
ocxo_real_time_error.csv
ocxo_real_frequency.csv
ocxo_real_timestamp.csv
```

优先级：

```text
time_error_s > frequency_hz > timestamp_count
```

如果只能提供 `timestamp_count`，请同时提供：

```text

```

## 10. 论文表述建议

如果采集链路满足高分辨率和高稳定参考源要求，可以写：

```text
采用高稳定参考源对 10 MHz OCXO 进行长时间相位时间误差采集，并基于重叠 Allan 偏差公式计算其频率稳定度。
```

如果仍使用 50 MHz FPGA 粗计数，应写：

```text
采用 FPGA 50 MHz 参考时钟对 10 MHz OCXO 输出进行时间戳采集，并对该测量链路下的 Allan 偏差进行分析。由于单周期仅对应 5 个参考时钟计数，短平均时间区域主要受时间量化误差影响，因此该结果用于验证 Allan 运算流程和测量链分辨率限制。
```
