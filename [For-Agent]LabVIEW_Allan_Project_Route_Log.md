# LabVIEW Allan 方差上位机工程路线日志

## 0. 当前工程定位

项目名称：基于 FPGA 频率稳定度测量系统的 LabVIEW 上位机设计

当前工程重点不是重新实现完整下位机通信系统，而是完成一个可演示、可测试、可交付的 Allan 方差分析上位机闭环。

当前主线：

```text
CSV/TDMS测试数据输入
→ 时间误差序列 x[k] 提取
→ Allan 方差 / Allan 偏差计算
→ 曲线显示
→ TDMS工程数据保存
→ TDMS结果回放
→ CSV复核文件导出
→ CSV复核回放
→ 性能统计与测试报告
```

核心设计思想：

```text
TDMS = 默认工程数据存储格式
CSV = 人工可读 / 论文复核 / MATLAB与Excel对比导出格式
```

## 3. 保存策略最终决策

不要把 CSV 保存和 TDMS 保存设计成完全平级按钮。

最终推荐：

```text
TDMS 作为默认工程数据存储格式；
CSV 作为人工复核导出格式。
```

原因：

| 维度             | TDMS            | CSV   | 结论           |
| -------------- | --------------- | ----- | ------------ |
| 文件体积           | 小               | 大     | TDMS更适合默认保存  |
| LabVIEW原生支持    | 强               | 一般    | TDMS更适合工程主链路 |
| 结构化程度          | Group/Channel清晰 | 依赖列顺序 | TDMS更稳       |
| 人工可读性          | 弱               | 强     | CSV适合复核导出    |
| MATLAB/Excel对比 | 一般              | 强     | CSV适合论文验证    |
| 工程扩展性          | 好               | 一般    | TDMS更适合长期扩展  |

最终链路：

```text
主链路：
开始计算
→ Allan结果显示
→ 保存工程数据TDMS
→ TDMS结果回放

辅助复核链路：
导出CSV复核文件
→ CSV复核回放
→ MATLAB/Excel/论文数据复核
```

## 6. 性能测试补充路线

性能测试优先采用 VI 内部计时逻辑，不只依赖 LabVIEW 自带性能分析器。

推荐使用：

```text
Tick Count(ms)
```

计时方法：

```text
Tick Count(ms) before
→ 功能模块
→ Tick Count(ms) after
→ 相减
→ /1000
→ 输出耗时_s
```

前面板新增显示控件 Indicator：

| 控件名              | 类型                | 说明           |
| ---------------- | ----------------- | ------------ |
| CSV读取耗时_s        | 数值显示控件 Indicator  | 输入CSV读取耗时    |
| Allan计算耗时_s      | 数值显示控件 Indicator  | Allan多m计算耗时  |
| TDMS保存耗时_s       | 数值显示控件 Indicator  | 保存TDMS耗时     |
| TDMS回放耗时_s       | 数值显示控件 Indicator  | 读取TDMS并显示耗时  |
| CSV导出耗时_s        | 数值显示控件 Indicator  | 导出CSV复核文件耗时  |
| 样本处理速率_pts_per_s | 数值显示控件 Indicator  | N / 总处理耗时    |
| 错误状态             | 错误簇显示控件 Indicator | error out 状态 |

优先补 4 个核心耗时：

```text
CSV读取耗时_s
Allan计算耗时_s
TDMS保存耗时_s
TDMS回放耗时_s
```

LabVIEW 自带工具：

```text
Profile Performance and Memory
```

用途：辅助观察 VI 执行耗时和内存占用。

定位：辅助工具，不作为主要测试报告来源。

| 多次运行稳定性 |       10次或30次 | 重复运行   | 错误码0  | 稳定    |

稳定性测试不强制做真实8小时采集，可采用：

```text
重复运行 10 / 30 次
记录错误码、成功次数、输出文件完整性、回放曲线一致性
```

论文表述：

```text
由于本设计采用离线测试数据验证上位机处理链路，稳定性测试采用重复运行方式进行，记录多次运行中的错误状态、输出文件完整性和回放一致性。
```

## 8. 不建议继续扩展的内容

当前不建议新增：

```text
真实串口高速接收
生产者消费者队列
环形缓存
真实丢包率统计
8小时真实连续采集
端到端硬件延迟
复杂数据库索引
更多文件格式
复杂另存为格式选择器
```

理由：

```text
这些会把工程拉回通用采集系统，偏离当前已经成型的 Allan 方差上位机闭环。
```

如果必须回应通信/缓存要求，可写为：

```text
系统接口层预留与下位机实时通信扩展能力，当前验证阶段采用CSV/TDMS文件作为上位机输入与结果管理方式。
```

